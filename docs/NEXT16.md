# Next.js 16 App Router Cheat Sheet

Sources: `node_modules/next/dist/docs/01-app/**`. Only what changed vs. older Next / is easy to get wrong.

## 1. Root layout & metadata

No change to the root layout API itself, but confirm the pattern (`03-file-conventions/layout.md`):

```tsx
// app/layout.tsx
export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

- Root layout **must** define `<html>`/`<body>`. Never hand-write `<title>`/`<meta>` in it — use the `metadata` export or `generateMetadata`.
- CHANGED: layouts do not rerender on navigation, so `params` is still a `Promise` (see §3) and layouts have **no access** to `searchParams` at all (use a Client Component + `useSearchParams` instead).
- CHANGED (16, cosmetic): Next no longer forces `scroll-behavior: auto` during navigation. If you rely on CSS `scroll-behavior: smooth` and want the old snap-to-top behavior back, add `data-scroll-behavior="smooth"` to `<html>` (`02-guides/upgrading/version-16.md`).

## 2. Server Actions

`03-api-reference/01-directives/use-server.md`, `02-guides/server-actions.md`, `02-guides/forms.md`

```ts
// app/actions.ts
'use server'
import { auth } from '@/lib/auth'

export async function createPost(formData: FormData) {
  const session = await auth()
  if (!session?.user) throw new Error('Unauthorized')
  const title = formData.get('title')
  // mutate + revalidate
}
```

- File-level `'use server'` (top of file) marks **every export** as a Server Function. Inline `'use server'` at the top of a function body works too (in Server Components only — Client Components cannot define one, only import/call one).
- Form usage — pass the function straight into `action`:
  ```tsx
  <form action={createPost}>...</form>
  ```
- Calling from a Client Component: import the action from a `'use server'` file and use it as `formAction`, or call it in an event handler (`onClick={() => createPost()}`), or pass it down as a prop.
- Extra args: `action.bind(null, userId)` — the bound arg arrives before `formData` in the signature: `async function updateUser(userId: string, formData: FormData)`.
- Validation errors / `useActionState`: signature gains a leading `prevState` param: `async function createUser(initialState: any, formData: FormData)`.
- CHANGED: the client **dispatches actions sequentially**, one at a time — do not `Promise.all()` multiple actions from the client; do parallel work inside one action instead.
- Every action is a public POST endpoint. Always re-check auth/ownership inside the action itself — never rely on the form only being rendered behind auth.

## 3. `params` / `searchParams` are Promises (CHANGED)

`03-api-reference/03-file-conventions/page.md`, `.../layout.md`

```tsx
// app/blog/[slug]/page.tsx
export default async function Page({
  params,
  searchParams,
}: {
  params: Promise<{ slug: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const { slug } = await params
  const { page = '1' } = await searchParams
}
```

- As of Next 16, **sync access to `params`/`searchParams` is fully removed** (15 had a deprecated compat shim; 16 does not) — `02-guides/upgrading/version-16.md` § "Async Request APIs (Breaking change)". Same applies to `cookies()`, `headers()`, `draftMode()`.
- In a Client Component page, unwrap with React's `use()`: `const { slug } = use(params)`.
- New `PageProps<'/blog/[slug]'>` / `LayoutProps<'/dashboard'>` global helper types give typed `params`/`searchParams` — generated by `next dev`/`build`/`next typegen`, no import needed.
- `searchParams` is request-time only and opts the route into dynamic rendering.

## 4. `revalidatePath` / `revalidateTag`

`03-api-reference/04-functions/revalidatePath.md`, `revalidateTag.md`

```ts
'use server'
import { revalidatePath, revalidateTag } from 'next/cache'

revalidatePath('/posts')                 // unchanged
revalidateTag('posts', 'max')            // CHANGED: 2nd arg now required
```

- CHANGED (16): `revalidateTag(tag)` single-arg form is **deprecated** and TS-errors; must pass a profile, `'max'` is the recommended stale-while-revalidate default. For immediate expire on webhooks, `revalidateTag(tag, { expire: 0 })`.
- New `updateTag(tag)` (Server Actions only, `next/cache`) gives immediate read-your-own-writes and re-renders the current route in the same response — prefer it over `revalidateTag` when the user must see their own change instantly.
- `revalidatePath('/product/[slug]', 'page')` needs the `type` param when the path has a dynamic segment; omit `type` for literal paths.

## 5. `redirect` / `useRouter`

`03-api-reference/04-functions/redirect.md`, `use-router.md`

```ts
import { redirect } from 'next/navigation'   // NOT next/router
redirect('/posts')   // 303 inside a Server Action, 307 elsewhere; call outside try/catch
```

```tsx
'use client'
import { useRouter } from 'next/navigation'   // NOT next/router
const router = useRouter()
router.push('/dashboard')
```

- Both live in `next/navigation`, never `next/router` (that's Pages Router only).
- `useRouter` no longer exposes `pathname`/`query`/`events` — use `usePathname()` / `useSearchParams()` instead.
- `redirect()` can't be called from a Client Component event handler — use `useRouter().push/replace` there.

## 6. `next/font` (Geist)

`01-getting-started/13-fonts.md` — no breaking API change, confirm the pattern:

```tsx
// app/layout.tsx
import { Geist } from 'next/font/google'
const geist = Geist({ subsets: ['latin'] })

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" className={geist.className}>
      <body>{children}</body>
    </html>
  )
}
```

Apply the font's `className` at the Root Layout level to scope it app-wide; fonts are otherwise scoped to whatever component uses them.

## 7. Other breaking defaults / config (from `02-guides/upgrading/version-16.md`)

- **Turbopack is default** for `next dev`/`next build` now — drop `--turbopack` flags; a custom `webpack()` config in `next.config` will fail the build unless you pass `--webpack`.
- **Node.js 20.9+ required**, TypeScript 5.1+.
- **`middleware.ts` → `proxy.ts`**: file and exported function both renamed (`export function middleware` → `export function proxy`); `edge` runtime not supported in `proxy` (stays `nodejs`).
- **`next lint` removed** — use ESLint/Biome CLI directly; `next build` no longer lints.
- **`serverRuntimeConfig`/`publicRuntimeConfig` removed** — use env vars (`NEXT_PUBLIC_*` for client-exposed ones).
- **Parallel routes**: every slot (`@modal`, etc.) now requires an explicit `default.js`, or the build fails.
- **`cacheLife`/`cacheTag`**: stable now, drop the `unstable_` prefix.
- **`images.domains`** deprecated → use `images.remotePatterns`; `next/legacy/image` deprecated → use `next/image`.
- PPR: use top-level `cacheComponents: true` in `next.config`, not `experimental.ppr` (removed).
